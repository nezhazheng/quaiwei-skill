# De AI Quotes

## Overview

清理AI生成文本中的典型痕迹，让文本更自然、更像人写的。AI生成的中文文本有两个明显特征：喜欢给各种词语加引号，以及在中英文之间加空格。这些习惯让文本一眼就能看出是AI写的。

## 处理规则

### 规则1：去除中文文本中英文单词周围的引号

AI在中文语境中提到英文单词时，几乎总会加上引号，这是典型的AI味。

处理前：
- 使用"Python"进行开发
- 这个"API"接口很好用
- 建议采用"React"框架
- 所谓的"best practice"

处理后：
- 使用Python进行开发
- 这个API接口很好用
- 建议采用React框架
- 所谓的best practice

匹配模式：中文引号（""或「」）包裹纯英文单词/短语（可含数字、点号、连字符、空格）时，去除引号保留内容。

### 规则2：去除对普通词语的不必要引号

AI喜欢对普通概念、常见术语加引号以示强调，但大多数情况下这些引号是多余的。

处理前：
- 这种"方法"很有效
- 实现了"自动化"部署
- 所谓的"最佳实践"
- 这是一种"创新"的方式

处理后：
- 这种方法很有效
- 实现了自动化部署
- 所谓的最佳实践
- 这是一种创新的方式

### 规则3：去除中文与英文之间的多余空格

AI在中文内容中插入英文单词时，习惯在英文前后各加一个空格。这不是中文排版的通行做法，反而是AI生成文本的明显标志。

处理前：
- 使用 Python 进行开发
- 这个 API 接口很好用
- 建议采用 React 框架
- 通过 HTTP 协议传输

处理后：
- 使用Python进行开发
- 这个API接口很好用
- 建议采用React框架
- 通过HTTP协议传输

匹配模式：中文字符与英文单词（含数字）之间的空格，当两侧分别是中文和英文时去除该空格。

#### 不应去除空格的情况

1. 英文单词之间的空格（如 best practice）→ 保留
2. 中文标点与英文之间的空格 → 保留
3. 句首或行首的缩进空格 → 保留
4. 代码块中的空格 → 保留

### 不应去除引号的情况

以下场景引号有实际语义作用，应保留：

1. **直接引语**：他说"这个方案可行" → 保留
2. **书名号替代**：引用文章标题、书名等 → 保留
3. **反讽或特殊含义**：这个"专家"连基础都不懂 → 保留（此处引号表达反讽）
4. **代码中的字符串字面量** → 保留
5. **引用他人原话或专有名词的正式定义** → 保留

判断标准：如果去掉引号后句意不变、不影响理解，则去除；如果引号承载了反讽、引用等语义功能，则保留。

## 工作流程

1. 读取目标文件内容
2. 逐行扫描，识别被引号包裹的内容和中英文之间的多余空格
3. 判断引号是否属于应去除的类别（英文单词周围的引号、普通词语的强调性引号）
4. 判断是否属于应保留的类别（直接引语、反讽、代码字符串等）
5. 对应去除的引号，删除引号符号，保留引号内的文本内容
6. 去除中文与英文之间的多余空格（跳过代码块）
7. 使用 Edit 工具逐处替换，确保修改可追溯

## 适用场景

- 用户明确要求去AI味、去引号
- 清理AI生成的文档、文章、README等
- 润色中文技术文档中的AI痕迹
